## Anomalies investigated

- **`lib/pq v1.12.3` (upstream historically topped out at v1.10.9):** NOT a fork/replace/proxy artifact. It is a genuine upstream tag — served by proxy.golang.org with sum.golang.org verification (`Origin github.com/lib/pq refs/tags/v1.12.3`), no `replace` directives, go.sum consistent. Upstream resumed tagging (v1.11.0–v1.12.3 series) and is active again. golang-migrate requires only v1.10.9; MVS selects v1.12.3 from the direct pin.
- **`envoyproxy/go-control-plane/envoy` + `google/flatbuffers` (indirect):** arrive via `xo/usql`'s **driver test packages** (`drivers.test` → spanner / aliyun-tablestore chains). They exist only in the module graph, are **never compiled into the sequa binary** (sequa imports only `usql/drivers/postgres`), and vanish if usql is ever dropped.
- **`go-git/go-billy/v5`:** a REAL build dep — `internal/query` → `xo/usql/handler` → go-billy (usql's filesystem abstraction). Compiled in.
- **`go.uber.org/atomic v1.9.0`:** a REAL build dep via `golang-migrate/migrate/v4/database/postgres`. Upstream frozen (superseded by `sync/atomic` types since Go 1.19) but stable and tiny.
- **`go mod tidy -diff`:** no output, exit 0 — module files fully tidy.
