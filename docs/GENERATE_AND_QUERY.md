# `sequa generate` & `sequa query`

Sequa's code-side verbs, both driven by the **migration-defined schema** (the
single source of truth) and a `sequa.yaml` config.

---

## `sequa generate` — type-safe Go from your schema + queries

Static, offline codegen (no database needed). The schema is parsed with
PostgreSQL's own parser (`pg_query_go` / `libpg_query`); the migrations are the
source of truth, so generated code never drifts from the applied schema.

> **cgo note:** `pg_query_go` is cgo — building anything that imports the
> codegen package requires a C compiler (gcc). The `migrate`/`query` paths stay
> pure-Go; only `generate` needs cgo.

### `sequa.yaml`

Place it at your project root (the analogue of `sqlc.yaml`):

```yaml
version: "1"
sql:
  - engine: postgresql                  # only postgresql in this milestone
    schema: internal/store/migrations   # the migrations dir (schema source)
    queries: internal/store/queries.sql # a .sql file OR a dir of *.sql (optional)
    gen:
      go:
        package: db                     # generated package name
        out: internal/store/db          # output directory
```

Run it:

```
sequa generate                 # uses ./sequa.yaml
sequa generate --config path/to/sequa.yaml
```

It writes `<out>/models.go` always, and `<out>/queries.go` when `queries` is set.

### Models

One struct per table, assembled by applying the up-migrations in order
(`CREATE TABLE`, `ALTER TABLE ADD/DROP COLUMN`, `DROP TABLE`):

```go
// Code generated by sequa. DO NOT EDIT.
package db

// Task maps a row of the "tasks" table.
type Task struct {
	ID        int64          `json:"id" db:"id"`
	Title     string         `json:"title" db:"title"`
	Done      bool           `json:"done" db:"done"`
	Note      sql.NullString `json:"note" db:"note"`        // nullable → sql.Null*
	CreatedAt time.Time      `json:"created_at" db:"created_at"`
}
```

Type mapping (Postgres → Go): `int8/bigserial`→`int64`, `int4/serial`→`int32`,
`bool`→`bool`, `text/varchar/uuid`→`string`, `float8`→`float64`,
`timestamptz/timestamp/date`→`time.Time`, `bytea/json/jsonb`→`[]byte`, `T[]`→`[]T`.
Nullable scalars use `sql.NullString`/`sql.NullInt64`/`sql.NullBool`/`sql.NullTime`/…
Table names are singularized + CamelCased (`tasks` → `Task`); columns CamelCased
with Go initialisms (`user_id` → `UserID`).

### Queries

Annotate each query with sqlc-style headers in your `queries` file:

```sql
-- name: GetUser :one
SELECT * FROM users WHERE id = $1;

-- name: ListByEmail :many
SELECT id, email FROM users WHERE email = $1;

-- name: CreateUser :one
INSERT INTO users (email, name) VALUES ($1, $2) RETURNING *;

-- name: DeleteUser :exec
DELETE FROM users WHERE id = $1;
```

| Command | Returns |
|---|---|
| `:one`  | `(Row, error)` via `QueryRowContext` |
| `:many` | `([]Row, error)` via `QueryContext` |
| `:exec` | `error` via `ExecContext` |

The result row is the **table model** for `SELECT *` / `RETURNING *`, or a
per-query `<Name>Row` struct for an explicit column list. Parameter Go types are
inferred from the column each `$N` binds to (in `WHERE`, `VALUES`, or `SET`).
Generated output:

```go
type DBTX interface { /* ExecContext/QueryContext/QueryRowContext */ }
func New(db DBTX) *Queries { return &Queries{db: db} }

func (q *Queries) GetUser(ctx context.Context, id int64) (User, error) { … }
func (q *Queries) ListByEmail(ctx context.Context, email string) ([]ListByEmailRow, error) { … }
func (q *Queries) CreateUser(ctx context.Context, email string, name sql.NullString) (User, error) { … }
func (q *Queries) DeleteUser(ctx context.Context, id int64) error { … }
```

`*sql.DB` and `*sql.Tx` both satisfy `DBTX`, so `db.New(conn)` just works.

### Supported query shapes (this milestone)

Single-table `SELECT` / `INSERT` / `UPDATE` / `DELETE`, with `*` or an explicit
column list, `RETURNING`, and params bound through `WHERE` / `VALUES` / `SET`.
Multi-table joins, expressions, and function results in the projection are not
yet inferred — those queries report a clear error so nothing silently generates
wrong types.

## `sequa query` — SQL client + REPL (embedded usql)

```
sequa query "postgres://user:pass@localhost/db?sslmode=disable"   # interactive REPL
sequa query --dsn "$DATABASE_URL"                                 # REPL (env/flag DSN)
sequa query --dsn "$DATABASE_URL" -c "SELECT * FROM tasks;"       # one-shot, prints a table
```

Powered by [xo/usql](https://github.com/xo/usql): psql-style result tables,
backslash meta-commands (`\dt`, `\d`, …) in the REPL, and DSN parsing via
`dburl`. The DSN may be a positional argument, `--dsn`, or `$DATABASE_URL`.
